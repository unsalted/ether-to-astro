# ether-to-astro v1.3.0

Suggested release type: `minor`

This release continues the post-1.2 cleanup work, with a small but explicit structured-payload correction in the rising-sign window output. ⚠️

## 1.3.0 (draft)

### API cleanup

- ⚠️ Breaking change: `getRisingSignWindows` structured window payloads now emit `durationMs` instead of `durationMinutes`
- ✅ `start` and `end` remain the authoritative machine-readable boundaries for rising-sign windows
- 💬 Human-readable rising-sign text output is unchanged

## Upgrade notes

- 🚨 Breaking change: `getRisingSignWindows` now uses `durationMs` in each serialized window object
- ❌ `durationMinutes` is no longer present in the structured payload
- 🔧 If you consume rising-sign window output programmatically, update any client code that reads or displays the old minute-based field

## Notes

- 🧭 This is an early lifecycle contract correction intended to keep structured data precise and leave lossy display formatting to presentation layers
